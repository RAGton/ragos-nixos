# Documentation Rules

Regras para manter a integridade da documentação do Kryonix.

- **docs/ é fonte canônica:** A documentação em `docs/` é a verdade para humanos e deve ser respeitada por agentes.
- **Roadmap separado do estado atual:** Funcionalidades em desenvolvimento devem estar em `ROADMAP.md` ou marcadas como `Status: Roadmap`.
- **docs/archive não é fonte primária:** Arquivos em `archive/` são para consulta histórica, não para definir comportamento atual.
- **Feature sem validação vai para ROADMAP:** Se uma feature não pode ser validada no ambiente atual, ela não pode ser marcada como pronta ou implementada.
