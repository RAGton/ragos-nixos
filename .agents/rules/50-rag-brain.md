# RAG & Brain Rules

Regras para interação com o cérebro técnico do Kryonix.

- **Grounding em docs canônicas:** O Brain/RAG deve responder baseando-se prioritariamente em documentação canônica (`docs/`).
- **ROADMAP não é implementação:** Não trate itens do Roadmap como funcionalidades prontas.
- **Archive é histórico:** Use `docs/archive/` para contexto histórico, não como verdade operacional atual.
- **Sinalizar falta de evidência:** Se não houver base documental ou de código para uma resposta, o agente deve informar explicitamente a falta de grounding.
