# Core Rules

Regras fundamentais para operação segura no Kryonix.

- **Menor mudança segura:** Sempre prefira a menor alteração que resolve o problema.
- **Não inventar feature:** Não implemente o que não foi pedido ou o que não tem base no código real.
- **Não declarar pronto sem evidência:** O status "Pronto" exige validação real e registro de evidência.
- **Preservar histórico útil:** Não apague documentação ou código histórico sem mover para `archive/`.
- **Rollback sempre que possível:** Tenha sempre um plano de retorno em caso de falha.
